# Story 2.4: BMAD install step

## Summary

Create a non-interactive BMAD installer module (`installer/src/bmad.js`) that orchestrates the `npx bmad-method install` command with the exact flags required by the Ralph Loop. The step runs under a visual spinner, is user-skippable during the install flow, and handles failures gracefully by printing the manual command and continuing non-fatally.

## Requirements

### Functional Requirements

**FR-5.1 Command Construction**
- Module exports a function that constructs and runs: `npx bmad-method install --modules core,bmm --tools claude-code --output-folder docs`
- Must include artifact path flags (--artifact-folder, --memory-folder) that match the Ralph Loop's documented directory structure
- Flags must be passed in this exact order (unit test will verify argv array)

**FR-5.2 Non-Interactive Execution**
- Uses native Node.js child_process (`execFile` or `spawn`) to invoke npx
- **No prompts or interaction** — the BMAD step must not present any wizard dialogs
- All BMAD configuration is provided via flags; stdin is not a tty

**FR-5.3 Visual Feedback (Spinner)**
- While BMAD is running, display a spinner (use the same spinner library as the rest of the installer — likely `@clack/prompts` which includes `spinner()`)
- Spinner message: something like "Installing BMAD modules..." or "Setting up BMAD..."
- Spinner is plain text in non-TTY mode (no ANSI escapes when piped)

**FR-5.4 Skippability**
- The calling code (`writer.js`) determines whether the BMAD step runs
- If the user declined BMAD during the wizard, this step is not called
- Module itself does not need to ask; the decision is made upstream in `writer.js`

**FR-5.5 Failure Handling (Non-Fatal)**
- If `npx bmad-method install` exits non-zero, **do not throw**
- Instead:
  - Stop the spinner
  - Print the exact manual command that the user should run: `npx bmad-method install --modules core,bmm --tools claude-code --output-folder docs [artifact flags]`
  - Include brief guidance: "BMAD installation failed. Run the above command to install manually, or proceed without BMAD and update later with `npx bmad-method update`."
  - Return success (not a throw)
- The installer continues to the next step

**FR-5.6 Stdout/Stderr Handling**
- Suppress BMAD's normal stdout (the verbose progress messages)
- If stderr contains error output, capture it for the failure message
- In TTY mode, display only the spinner
- In non-TTY mode, show a brief line like "Installing BMAD..." before the result

## Implementation Steps

### Step 1: Create `installer/src/bmad.js`

Create the module with the following structure:

1. **Imports:**
   - `{ execFile } from 'node:child_process'`
   - `{ promisify } from 'node:util'`
   - `{ spinner } from '@clack/prompts'` (if available; otherwise use a plain-text progress marker)
   - `{ isUnicodeSupported } from '@clack/prompts'` (for ANSI check)

2. **Constants:**
   ```javascript
   const BMAD_COMMAND = 'npx';
   const BMAD_ARGS = [
     'bmad-method',
     'install',
     '--modules', 'core,bmm',
     '--tools', 'claude-code',
     '--output-folder', 'docs',
     '--artifact-folder', 'docs',
     '--memory-folder', 'docs/_bmad/_memory'
   ];
   ```

3. **Main function `installBmad(options = {})`:**
   - Accept `options.spinner` (external spinner instance) and `options.isTTY` (boolean)
   - Create a spinner if not provided (or use a no-op for non-TTY)
   - Call `execFile(BMAD_COMMAND, BMAD_ARGS, { stdio: ['ignore', 'pipe', 'pipe'] })`
   - Start spinner with message "Installing BMAD modules..."
   - On success: stop spinner, return `{ success: true }`
   - On failure: stop spinner, console.log the manual command, return `{ success: true }`
   - Never throw

4. **Manual command printer:**
   - Function `printManualCommand()` that outputs:
     ```
     BMAD installation failed. Run this command to install manually:
     
     npx bmad-method install --modules core,bmm --tools claude-code --output-folder docs --artifact-folder docs --memory-folder docs/_bmad/_memory
     
     Or proceed without BMAD now and update later with: npx bmad-method update
     ```

## Edge Cases & Gotchas

**Edge Case 1: BMAD not found**
- If `npx` itself fails (e.g., npm is not installed), execFile will throw
- Catch this in the try/catch and treat as a failure (non-fatal, print manual command)

**Edge Case 2: Network failure during npx download**
- BMAD might fail if the npm registry is unreachable
- Treated as a failure; manual command is printed
- User can retry manually when network is available

**Edge Case 3: User cancels the install before BMAD step**
- If the user presses Ctrl-C during the wizard, nothing runs (no issue)
- If they cancel after confirming the install but before BMAD runs, BMAD is skipped (correct behavior)

**Edge Case 4: BMAD already installed**
- Re-running the installer when BMAD is already present in `docs/`
- The `npx bmad-method install` command is idempotent; running it again should succeed or be a no-op
- No special handling needed; treat as success

**Edge Case 5: Different artifact paths**
- The story hardcodes `docs`, `docs/_bmad/_memory`
- If a future story allows the user to customize the docs path, update the artifact flags to use the user's choice
- For now, assume `docs` is always the target

**Edge Case 6: Spinner in piped/CI environment**
- @clack/prompts should detect non-TTY automatically
- Verify that `process.stdout.isTTY` is false and spinner is suppressed
- Output should be plain text, no ANSI codes

**Edge Case 7: Windows/WSL considerations**
- `execFile` uses the OS-native spawn, so cross-platform should work
- `npx` is available on Windows (via Node.js PATH)
- No special handling needed; @clack/prompts handles Windows terminal detection
