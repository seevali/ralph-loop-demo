## Acceptance criteria

1. **Update detection** — `existing-install` classification (manifest present) routes to `detectUpdate`, which logs `installed vX → available vY`. ✓ (`bin/ralph.js:83-107`)
2. **Classification honored** — `executeUpdate` skips every `user-owned` file via the absolute `getOwnership()` guard (`writer.js:631`), independent of the delta or decisions; installer-owned files are written. ✓
3. **Conflict resolution UI** — three options (keep/take/backup) offered interactively; `--update-conflicts` honored non-interactively; invalid values return `{succeeded:false}` and the CLI exits non-zero before any write. ✓ (`updateConflictResolver.js:25-31`, `bin/ralph.js:84-89`)
4. **User mods preserved byte-identically** — E2E asserts a user-owned file is byte-identical post-update; a companion E2E confirms the unmodified installer-owned file is replaced. ✓
5. **Backup hygiene** — `.backup` written only on a `backup` decision; E2E confirms no backup is created for `keep`. ✓
6. **Non-interactive consistency** — non-TTY / `--yes` / `--force` use `--update-conflicts` or default `keep`; invalid flag fails non-zero before writes. ✓
7. **Manifest update** — `writeUpdateManifest` updates `version` + checksums for written files, preserves `ownership`, `wizardAnswers`, `installedAt`, `targetClass`. The deliberate "keep retains prior installer checksum" behavior (so the next run re-detects the conflict) is sound and test-covered. ✓
