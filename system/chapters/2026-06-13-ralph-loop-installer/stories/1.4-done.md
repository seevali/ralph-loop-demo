## Key Implementation Decisions

### 1. Dependency injection instead of module-level mocking

The story spec suggested two testability approaches: `Module.prototype.require` monkey-patching or exporting a function that accepts a `spawnFn` parameter. Neither fits cleanly with ESM (no `require`). Instead, `preflight()` accepts an `opts` object with three injectable fields:

- `opts.nodeVersion` — replaces `process.version`
- `opts.platform` — replaces `process.platform`
- `opts.commandExists` — `(cmd: string) => boolean`, replaces the `execSync` wrapper

Similarly, `renderChecklist()` accepts `{ isTTY, noColor, writeFn }` for full control in tests. This approach requires zero test-framework mocking infrastructure and works cleanly with ESM.

### 2. `preflight()` never calls `process.exit()`

The module returns the results object; the caller decides exit codes. This is documented in JSDoc on `preflight()`:
- `results.node.status === 'fail'` → caller exits with code 1
- `results.bash.status === 'fail'` → caller exits with code 2

The tests verify the status values, not `process.exit()` calls.

### 3. NO_COLOR + TTY split

When `isTTY=true && noColor=false`: glyph output with picocolors.  
When `isTTY=true && noColor=true`: glyph output without ANSI codes (picocolors bypassed entirely).  
When `isTTY=false`: plain key–value text format, no glyphs, no ANSI codes.

This matches the NO_COLOR spec: "omit color even on TTY" while still showing glyphs (since glyphs are not color, they are structure).

### 4. Package.json test glob updated

The original test script was `node --test 'test/**/*.test.js'`. Updated to `node --test 'src/**/*.test.js' 'test/**/*.test.js'` so tests colocated beside source files are auto-discovered, matching the story's instruction for `installer/src/preflight.test.js`.

### 5. claude check status

AC4 says "both are warnings, no block" — interpreted as: present → `pass`, absent → `warn`. Neither case is `fail`. The phrase "both are warnings" means neither case blocks installation, not that both map to status `warn`.
