## Key Implementation Decisions

### ESM-only module

The package uses `"type": "module"` in `installer/package.json` (established by Story 1.2). `classify.js` is therefore an ES module using `import` syntax, not CommonJS `require()`. The verification step 4 in the story spec uses `require()`; that was adapted to `node --input-type=module -e "import ..."` for manual testing.

### Symlink resolution with `fs.realpath`

`fs.realpath(absolutePath)` handles both existence checking and symlink resolution in one call — it throws `ENOENT` for missing paths and returns the real path for symlinks. A subsequent `fs.stat` verifies the resolved path is a directory. `projectRoot` in the return value is always the real path.

### Signal collection order

`.ralph` is pushed first (manual, when `hasManifest === true`), then the readdir loop adds project signals in directory-listing order. This means `.ralph` always appears at index 0 of the signals array for existing-install results, which is predictable for callers (Story 2.3).

### `checkManifest` uses `fs.access`

`fs.access` with default `F_OK` is the lightest existence check — no stat overhead, no reads. It checks for `.ralph/manifest.json` only; a `.ralph` directory without the manifest does not trigger `existing-install`.

### No `.ralph` in `FILE_SIGNALS`

`.ralph` is not in the `FILE_SIGNALS` set. It is added manually when `hasManifest === true`. This prevents double-counting if a `.ralph` directory also appears in the `readdir` results.

## Test Cases Implemented (9 total)

| # | Scenario | Assertion |
|---|----------|-----------|
| 1 | Empty directory | `type === 'empty'`, `signals.length === 0`, `hasManifest === false` |
| 2 | Node project (`package.json` + `.git/` dir) | `type === 'existing-project'`, signals include both |
| 3 | Python project (`pyproject.toml` + `.git/` dir) | `type === 'existing-project'`, signals include `pyproject.toml` |
| 4 | Ralph install (`.ralph/manifest.json` + `package.json`) | `type === 'existing-install'`, `hasManifest === true`, signals include `.ralph` |
| 5 | Partial/ambiguous install (manifest present, no loop script) | `type === 'existing-install'` — manifest takes priority |
| 6 | Multiple signals (`.git/`, `package.json`, `Gemfile`) | All three appear in signals |
| 7 | Non-existent path | Throws with "not found" in message |
| 8 | Symlink to directory | `projectRoot` equals `fs.realpath(realDir)` |
| 9 | `.sln` file (Visual Studio) | `type === 'existing-project'`, signals include `MyApp.sln` |
