## Key Implementation Decisions

1. **Unescaped content in markdown files.** The script uses `\"` and `` \` `` for bash string escaping inside the `common` double-quoted assignment. The extracted markdown files contain the unescaped characters (real backticks, real double-quotes) as shown in the story spec's content blocks. The loader (story 1.2) reads the files as-is; no unescaping step is needed.

2. **`{{CHECKPOINT_CMD}}` placeholder.** The single bash variable `${CHECKPOINT_CMD}` in the script's `common` block was replaced with `{{CHECKPOINT_CMD}}` (double braces) as specified. This is the only placeholder; no other substitutions are permitted in Layer 3 files.

3. **SM and Dev overlays are placeholder comments, not empty files.** A comment line was used instead of a zero-byte file to signal intent to future editors: these files exist for Layer 3 overlay additions but currently have no content beyond the common layers.

4. **Review fallback is plain text only.** The `bmad-fallbacks/review.md` contains only `You are the BMAD Code Review agent.` — not the `# Agent Persona` header that appears in the script's `review_header` variable. The header wrapper is assembled by the loader (story 1.2), not stored in the fallback file. This matches the pattern of the SM and Dev fallbacks.
