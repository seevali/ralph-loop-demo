## Key Implementation Decisions

**Handler placement:** The dry-run block is placed at the top-level of the script, after the `main()` function definition (line ~1862) and before the `main` call. This is the earliest point where all prerequisites are satisfied: `REPO_ROOT` is resolved, BMAD personas are loaded into `AGENT_*_PERSONA` globals, and `load_prompt_layers()` is defined. Placing it here ensures `claude` is never invoked (it's only called inside `run_claude()`, inside `main()`).

**Variable naming:** Since the handler runs at top-level (not inside a function), `local` cannot be used. Variables are prefixed `_dryrun_` to avoid shadowing any existing globals.

**Output capture:** `_dryrun_prompt=$(load_prompt_layers "$_dryrun_role") || _dryrun_rc=$?` — stdout is captured (the assembled prompt); stderr (error messages from `load_prompt_layers`) passes through directly to the terminal. This keeps error messages out of the prompt output.

**Partial-failure semantics:** All three roles are attempted regardless of individual failures. `_dryrun_failed` accumulates failures; the exit code reflects whether any role failed. This lets contributors see all errors in one run.

**Checkpoint form:** `$LOOP_SCRIPT --dry-run-prompts >/dev/null` uses the absolute path variable (already set in `ralph-loop-system.sh`) for consistency with the existing `bash -n $LOOP_SCRIPT` pattern.

**`printf '%s\n'` instead of `echo`:** Used to print the assembled prompt to avoid misinterpreting backslash sequences that may appear in the prompt content.
