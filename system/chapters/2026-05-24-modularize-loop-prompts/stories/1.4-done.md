## Key Implementation Decisions

### 1. Baseline capture via awk extraction, not full source

Sourcing `ralph-loop.sh` directly is impractical: the script calls `main` unconditionally on its last line, which would attempt to run the loop. Instead, `capture_baseline()` builds a minimal extractor script by:

1. Writing a static header that sets `REPO_ROOT`, stubs `log_info`/`log_dim`/`log_warn`, and loads personas using the same logic as `ralph-loop.sh` lines 336–358.
2. Appending the `build_system_prompts()` function body extracted with awk (`/^build_system_prompts\(\)/{found=1} found{print} found && /^\}$/{exit}`).
3. Calling the function and writing the three `SYSTEM_PROMPT_*` variables to output files via `OUTDIR` env var.

This reproduces the exact output of `build_system_prompts()` without running the loop.

### 2. Layered output parsed from `--dry-run-prompts` stdout

`--dry-run-prompts` emits ANSI-escaped log lines to stdout before the content sections. Since log lines never start with `===`, standard awk section extraction (`/^=== SM ===$/{f=1;next}` etc.) ignores them automatically — no ANSI stripping needed.

### 3. Additional filter for the known REVIEW architectural difference

`build_system_prompts()` prepends a hardcoded sentence to the REVIEW prompt when `AGENT_REVIEW_PERSONA` is set:
> "You are the BMAD Code Review agent. Apply the adversarial review approach (Blind Hunter + Edge Case Hunter + Acceptance Auditor) and triage methodology. The full BMAD review workflow is provided below."

This sentence is intentionally absent from the new layered architecture (the SKILL.md provides its own role description). Without filtering it, the REVIEW role would report 1 missing line and exit 1. The script adds this specific string to the insignificance filter with an inline comment explaining the rationale — analogous to the existing `# Agent Persona` stub heading filter specified in the story.

### 4. `grep -qF -- "$line"` for safe fixed-string matching

Prompt lines starting with `-` (markdown list items) would be misinterpreted as grep options without the `--` separator. Using `grep -qF -- "$line"` handles all line content safely.
