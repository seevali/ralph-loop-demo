## Key Implementation Decisions

### 1. Insertion point: after the BMAD loader, before `build_system_prompts()`

The function is defined at the boundary between the BMAD loader block (lines ~317–353) and the "Build cached system prompts" section. This placement:
- Keeps it in the Helpers region (readable alongside `log_info`, `fadd`, etc.)
- Ensures `$AGENT_*_PERSONA` variables are globally assigned before the function is ever called (the BMAD loader runs at script top-level before any function call)
- Positions it before `build_system_prompts()` so story 1.5's wiring is a short, local edit

### 2. `&` escape in bash `${//}` substitution

`CHECKPOINT_CMD` contains `&&` (e.g. `cd src && npm run build && npm test`). In bash's parameter substitution `${var//pat/rep}`, an `&` in the replacement string is treated as a backreference to the matched text (same as sed). Without escaping, `&&` would expand to the matched pattern twice, corrupting the output.

Fix: pre-escape `&` before substitution:
```bash
local escaped_cmd="${CHECKPOINT_CMD//&/\\&}"
result="${result//\{\{CHECKPOINT_CMD\}\}/$escaped_cmd}"
```

This is the only non-obvious implementation detail. Everything else follows the spec directly.

### 3. Function outputs to stdout only

`load_prompt_layers()` never writes to global variables — output goes to stdout, captured by the caller with `$(load_prompt_layers "role")`. This matches the spec and keeps the function pure for testing. Story 1.5 will assign the captured output to `SYSTEM_PROMPT_*` globals.

### 4. Error semantics: fail fast, never silent

Every missing file exits with return code 1 and writes to stderr. The function never produces partial output — either all three layers assemble cleanly or it returns an error. This is intentional: a truncated system prompt would silently degrade loop quality.
