## Summary

Add a `--dry-run-prompts` CLI flag to `scripts/ralph-loop.sh` that calls `load_prompt_layers()` for each role (SM, Dev, Review), prints the three resolved system prompts to stdout with clear delimiters, and exits 0 before any `claude` invocation. This flag serves as the safety harness for validating prompt composition in story 1.4 and as a debugging tool for future contributors.

### Phase 2: Implement the dry-run logic

**File: `scripts/ralph-loop.sh`**

Replace the Phase 1 placeholder with:

```bash
if [[ "${1:-}" == "--dry-run-prompts" ]]; then
    # Dry-run mode: load and print resolved prompts for all roles, then exit
    
    # Validate that load_prompt_layers() is available
    if ! declare -f load_prompt_layers &>/dev/null; then
        echo "Error: load_prompt_layers() not found" >&2
        exit 1
    fi
    
    # Define the three roles to process
    local roles=("sm" "dev" "review")
    local failed=0
    
    for role in "${roles[@]}"; do
        echo "=== $(echo "$role" | tr '[:lower:]' '[:upper:]') ==="
        
        # Call load_prompt_layers() and capture output
        if ! prompt_text=$(load_prompt_layers "$role" 2>&1); then
            echo "Error loading prompts for role '$role':" >&2
            echo "$prompt_text" >&2
            failed=1
        else
            # Print the prompt
            echo "$prompt_text"
        fi
        
        # Blank line between prompts for readability
        echo ""
    done
    
    # Exit with appropriate code
    if [[ $failed -eq 1 ]]; then
        exit 1
    fi
    exit 0
fi
```

**Implementation notes:**
- The loop processes roles in order: SM, Dev, Review (matches typical agent order)
- Headers use uppercase role names: `=== SM ===`, `=== DEV ===`, `=== REVIEW ===`
- Each prompt is followed by a blank line for visual separation
- Errors from `load_prompt_layers()` are captured and printed to stderr with context
- The function checks that `load_prompt_layers()` exists (safety against undefined functions)
- Exit code is 1 if any role failed, 0 if all succeeded
- All error output goes to stderr; only the prompts go to stdout

**Critical safeguards:**
- Use `2>&1` to capture stderr from `load_prompt_layers()` (if it outputs errors)
- Use `declare -f` to validate the function exists before calling it
- Avoid invoking `claude` anywhere in this code path

## Gotchas & Guardrails

1. **Don't invoke `claude`:** This flag must exit before any agent call. If you accidentally call `run_claude()` or similar, the story is not done.

2. **Preserve script semantics:** Do not modify `build_system_prompts()`, loop retry logic, multi-model routing, or any other existing loop behavior. This story only adds a new flag; it leaves the main loop untouched.

3. **Error messages to stderr:** All errors (missing files, function failures) should go to stderr. Only the three prompts go to stdout. This ensures `--dry-run-prompts >/dev/null` suppresses output correctly.

4. **Function signature for `load_prompt_layers()`:** If the actual function signature differs from the assumption above, adjust the implementation. The key requirement is: call it for each role and capture its output.

5. **Bash compatibility:** The script uses `bash -euo pipefail`. Ensure the new code follows this: use `[[  ]]` for conditionals, quote all variable expansions, use local variables in functions.

6. **Exit 0 on success, 1 on failure:** The exit code signals whether prompt resolution succeeded. Story 1.4 will rely on this exit code to detect problems.

7. **Don't modify `scripts/prompts/` files:** They are read-only from this story's perspective. Any content changes belong in story 1.1 or later.

8. **System Track guardrails (from system/CLAUDE.md):**
   - You may modify `scripts/ralph-loop.sh` (loop modifying itself)
   - You may modify `system/ralph-loop-system.sh` (system track checkpoint)
