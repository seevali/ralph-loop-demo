## Key Implementation Decisions

### 1. Rewrite of `build_system_prompts()`

Replaced the entire function body (the `common` heredoc, SM/Dev/Review fallback and persona-wrapping conditionals, Review Standards section — ~126 lines) with three `load_prompt_layers()` calls plus the existing logging lines. The logging block (`log_info` + three `log_dim` byte-count lines) is preserved verbatim.

### 2. Verification script update — necessary deviation from story spec

The story spec stated that `verify-semantic-equivalence.sh` "remains unchanged." However, after the refactor `build_system_prompts()` calls `load_prompt_layers()`, so the `capture_baseline` function's awk extraction would produce a script missing `load_prompt_layers`, causing it to fail with an undefined-function error.

**Fix:** Updated `capture_baseline` to loop over `[load_prompt_layers, build_system_prompts]` and run one awk extraction per function. Each uses the same pattern (`/^<name>\(\)/…/^\}$/{exit}`) as the original. This is a mechanical extension, not a logic change — the verification semantics are unchanged.
