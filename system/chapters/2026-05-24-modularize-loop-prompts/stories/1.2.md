# Story 1.2: Add `load_prompt_layers()` helper

**Epic:** [modularize-loop-prompts](../epics/modularize-loop-prompts.md)
**Chapter:** [2026-05-24-modularize-loop-prompts](../)
**Status:** review
**Story ID:** 1.2

## Summary

Add a new bash function `load_prompt_layers(role)` to `scripts/ralph-loop.sh` that assembles cached system prompts from three layers: execution-context override (Layer 1), live BMAD persona (Layer 2), and demo-specific rules (Layer 3). The function reads from the external MD files created in story 1.1, applies the `{{CHECKPOINT_CMD}}` substitution, and returns the assembled string. This function is the foundation for stories 1.3–1.5 but is NOT wired into `build_system_prompts()` yet — it stands alone, ready to be tested and verified before integration.

---

## Acceptance Criteria

1. **Function defined.** A bash function named `load_prompt_layers()` is defined in `scripts/ralph-loop.sh`, callable with a role argument (one of `sm`, `dev`, `review`).

2. **Layer 1 reads correctly.** Function reads `scripts/prompts/common/execution-context.md` and uses it as Layer 1 of the output.

3. **Layer 2 uses loaded BMAD persona.** Function uses the `$AGENT_<ROLE>_PERSONA` variable (already loaded by the existing BMAD loader, lines ~323–353) as Layer 2. For role `dev`, it uses `$AGENT_DEV_PERSONA`; for `sm`, `$AGENT_SM_PERSONA`; for `review`, `$AGENT_REVIEW_PERSONA`.

4. **Layer 2 fallback.** If the BMAD persona variable is empty (BMAD files not found or missing), function falls back to `scripts/prompts/bmad-fallbacks/<role>.md` and reads that file instead.

5. **Layer 3 reads correctly.** Function reads `scripts/prompts/common/project-conventions.md` and appends `scripts/prompts/<role>/overlay.md` to form Layer 3.

6. **Layers concatenated with separators.** The three layers are concatenated with `\n\n---\n\n` (two newlines, a markdown separator line, two newlines) as the joiner.

7. **Checkpoint substitution.** Function applies a single text substitution to the result: replaces all occurrences of `{{CHECKPOINT_CMD}}` with the value of `$CHECKPOINT_CMD` (a bash variable containing the checkpoint command string).

8. **Returns string to stdout.** Function outputs the assembled prompt string to stdout (via `echo` or `cat`). The caller captures it (e.g., `result=$(load_prompt_layers "dev")`).

9. **Syntax validation.** `bash -n scripts/ralph-loop.sh` passes with no errors.

10. **No behavior change yet.** `build_system_prompts()` remains unchanged — it still builds prompts using the inline `common` heredoc and the BMAD persona variables. The new function coexists in the script but is not called by any existing function.

## Technical Context

### Three-Layer Composition (from chapter plan §"Recommended path")

Each system prompt is assembled from three layers to balance BMAD liveness with loop robustness:

- **Layer 1: Execution Context Override** (`scripts/prompts/common/execution-context.md`)
  - Stable repo-local text (~9 lines).
  - Contains "do not greet, skip On Activation, do not HALT" rules that override BMAD's interactive instructions.
  - Order matters: Layer 1 first ensures these rules win.

- **Layer 2: BMAD Persona**
  - Live-loaded from `.claude/skills/<skill>/SKILL.md` (already done by lines ~323–353).
  - May be empty if BMAD is not installed or the role file is missing.
  - When empty, falls back to `scripts/prompts/bmad-fallbacks/<role>.md`.

- **Layer 3: Demo-Specific Rules**
  - Two files concatenated: `scripts/prompts/common/project-conventions.md` + `scripts/prompts/<role>/overlay.md`.
  - Contains stack-specific rules (React/Vite/TS) and role-specific standards (review pass/block criteria).
  - Order matters: Layer 3 last ensures these rules are the freshest instruction the model sees.

See `scripts/prompts/README.md` (created in story 1.1) for the full design rationale.

### Placeholder Substitution

The only templated value allowed in Layer 3 is `{{CHECKPOINT_CMD}}` (literal string with double braces, not bash `${}`). The loader replaces it with `$CHECKPOINT_CMD` (the bash variable set at line ~61 or overridden via `--checkpoint` CLI flag).

**Critical for prompt cache:** Only values that remain constant for the lifetime of a run should be substituted in the system prompt. `CHECKPOINT_CMD` is set once at startup; per-invocation values (story IDs, etc.) must stay in user prompts. See `scripts/prompts/README.md` §"The `{{CHECKPOINT_CMD}}` Placeholder" for the rationale.

### BMAD Persona Loading (Already Done)

Lines ~323–353 load the BMAD persona files into global variables:
- `AGENT_SM_PERSONA` from `.claude/skills/bmad-create-story/SKILL.md`
- `AGENT_DEV_PERSONA` from `.claude/skills/bmad-dev-story/SKILL.md`
- `AGENT_REVIEW_PERSONA` from `.claude/skills/bmad-code-review/SKILL.md` plus concatenated `steps/*.md` files

These variables are already set before `load_prompt_layers()` is called. The new function simply reads them.

### Files Created by Story 1.1

All paths assume these files exist (created by story 1.1):
- `scripts/prompts/README.md` — explains the loader and 3-layer model
- `scripts/prompts/common/execution-context.md` — Layer 1
- `scripts/prompts/common/project-conventions.md` — Layer 3 shared
- `scripts/prompts/sm/overlay.md` — Layer 3 SM-specific
- `scripts/prompts/dev/overlay.md` — Layer 3 Dev-specific
- `scripts/prompts/review/overlay.md` — Layer 3 Review-specific (contains Review Standards + UPSTREAM_FIX_REQUIRED format)
- `scripts/prompts/bmad-fallbacks/{sm,dev,review}.md` — fallback personas

### Step 2: Define `load_prompt_layers()` function

Add the function definition with the following signature and structure:

```bash
load_prompt_layers() {
  local role="$1"
  [[ -z "$role" ]] && { echo "ERROR: load_prompt_layers requires a role argument (sm, dev, review)" >&2; return 1; }

  local layer1 layer2 layer3 result

  # Layer 1: Execution Context Override (stable, repo-local)
  layer1="$(cat "$REPO_ROOT/scripts/prompts/common/execution-context.md" 2>/dev/null)"
  [[ -z "$layer1" ]] && { echo "ERROR: Layer 1 file not found: $REPO_ROOT/scripts/prompts/common/execution-context.md" >&2; return 1; }

  # Layer 2: BMAD Persona (live from .claude/skills/, or fallback to repo-local)
  case "$role" in
    sm)     layer2="$AGENT_SM_PERSONA" ;;
    dev)    layer2="$AGENT_DEV_PERSONA" ;;
    review) layer2="$AGENT_REVIEW_PERSONA" ;;
    *)      echo "ERROR: Unknown role '$role'. Expected one of: sm, dev, review" >&2; return 1 ;;
  esac

  # If BMAD persona is empty, fall back to repo-local fallback
  if [[ -z "$layer2" ]]; then
    layer2="$(cat "$REPO_ROOT/scripts/prompts/bmad-fallbacks/${role}.md" 2>/dev/null)"
    [[ -z "$layer2" ]] && { echo "ERROR: No BMAD persona and fallback file not found: $REPO_ROOT/scripts/prompts/bmad-fallbacks/${role}.md" >&2; return 1; }
    log_info "load_prompt_layers($role): using inline fallback (BMAD persona not found)"
  fi

  # Layer 3: Demo-Specific Rules (stable, repo-local)
  local layer3_common layer3_overlay
  layer3_common="$(cat "$REPO_ROOT/scripts/prompts/common/project-conventions.md" 2>/dev/null)"
  [[ -z "$layer3_common" ]] && { echo "ERROR: Layer 3 common file not found: $REPO_ROOT/scripts/prompts/common/project-conventions.md" >&2; return 1; }
  
  layer3_overlay="$(cat "$REPO_ROOT/scripts/prompts/${role}/overlay.md" 2>/dev/null)"
  [[ -z "$layer3_overlay" ]] && { echo "ERROR: Layer 3 overlay file not found: $REPO_ROOT/scripts/prompts/${role}/overlay.md" >&2; return 1; }
  
  layer3="${layer3_common}

${layer3_overlay}"

  # Assemble the three layers with separators
  result="${layer1}

---

${layer2}

---

${layer3}"

  # Substitute {{CHECKPOINT_CMD}} placeholder (whitelisted only)
  # Double braces distinguish placeholder syntax from bash variable syntax
  result="${result//\{\{CHECKPOINT_CMD\}\}/$CHECKPOINT_CMD}"

  # Output to stdout
  echo "$result"
}
```

## Edge Cases & Gotchas

### 1. **File not found errors**
   - If any layer file is missing, the function logs an error to stderr and returns exit code 1.
   - The function does NOT create missing files or silently skip them.
   - Test this by moving a file and calling `load_prompt_layers "dev"` to confirm the error.

### 2. **BMAD persona missing, fallback missing too**
   - If `$AGENT_DEV_PERSONA` is empty AND `scripts/prompts/bmad-fallbacks/dev.md` is missing, the function returns error code 1.
   - This is the safety net: the function will never produce incomplete output.

### 3. **Double-brace substitution gotcha**
   - The substitution uses bash parameter expansion: `"${result//\{\{CHECKPOINT_CMD\}\}/$CHECKPOINT_CMD}"`.
   - The double braces `{{` and `}}` in the source file must be literal; bash is case-sensitive and exact.
   - Test: call `load_prompt_layers "dev"` and pipe to `grep "CHECKPOINT_CMD"` to ensure the substitution happened.

### 4. **Newline handling**
   - Layers are joined with exactly `\n\n---\n\n` (two newlines, separator, two newlines).
   - Bash echo with `-n` does not add trailing newline; ensure the assembled string is clean.
   - Each layer file should end with a newline (standard Markdown practice). If a file does not, the separator may be malformed.

### 5. **Variable scope**
   - `load_prompt_layers()` reads `$REPO_ROOT` (set at line ~177) and `$AGENT_*_PERSONA` (set by the BMAD loader).
   - These must be set before the function is called. Test this in isolation by verifying the function is called after both the path-resolution block and the BMAD loader.

### 6. **Function does not write files**
   - `load_prompt_layers()` only reads and writes to stdout. It does NOT assign to global variables like `SYSTEM_PROMPT_SM`.
   - The caller must capture the output: `result=$(load_prompt_layers "sm")` or pipe it.
   - Story 1.5 (which wires the function into `build_system_prompts()`) will handle the variable assignment.
